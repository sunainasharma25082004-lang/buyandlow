# BuyLow — Render Deployment (3 Services)

Deploy **backend**, **store**, and **admin** as separate Render services.

## 1. MongoDB Atlas (free)

1. Create cluster at [mongodb.com/atlas](https://www.mongodb.com/atlas)
2. Database Access → create user + password
3. Network Access → allow `0.0.0.0/0` (or Render IPs)
4. Connect → copy connection string → replace `<password>` and set db name `buylow`

## 2. Push to GitHub

```bash
git add .
git commit -m "Prepare BuyLow for Render deployment"
git push origin main
```

## 3. Create services on Render

**Option A — Blueprint (recommended)**  
Render Dashboard → **New** → **Blueprint** → connect repo → uses `render.yaml`.

**Option B — Manual**  
Create 3 services with settings from `render.yaml`.

## 4. Environment variables (after first deploy)

Replace URLs with your actual Render URLs.

### buylow-api (Web Service)

| Variable | Example |
|----------|---------|
| `MONGODB_URI` | `mongodb+srv://...` |
| `API_BASE_URL` | `https://buylow-api.onrender.com` |
| `CLIENT_URL` | `https://buylow-store.onrender.com,https://buylow-admin.onrender.com` |
| `ADMIN_PASSWORD` | strong password |
| `RAZORPAY_KEY_ID` | optional |
| `RAZORPAY_KEY_SECRET` | optional |
| `SERVE_CLIENT` | `false` |

`JWT_SECRET` is auto-generated by blueprint.

### buylow-store (Static Site)

| Variable | Example |
|----------|---------|
| `VITE_API_URL` | `https://buylow-api.onrender.com/api` |
| `VITE_GOOGLE_CLIENT_ID` | `123456789.apps.googleusercontent.com` |

### Google Sign-In setup

1. [Google Cloud Console](https://console.cloud.google.com/apis/credentials) → Create OAuth **Web client**
2. **Authorized JavaScript origins:** `http://localhost:5173`, `https://buylow-store.onrender.com`
3. Copy Client ID → set `GOOGLE_CLIENT_ID` on **buylow-api** and `VITE_GOOGLE_CLIENT_ID` on **buylow-store** (same value)
4. Redeploy store + API after adding vars

### buylow-admin (Static Site)

| Variable | Example |
|----------|---------|
| `VITE_API_URL` | `https://buylow-api.onrender.com/api` |
| `VITE_STORE_URL` | `https://buylow-store.onrender.com` |

> After changing `VITE_*` vars, **redeploy** the static sites (rebuild required).

## 5. Seed database (first time)

Locally with production URI:

```bash
cd server
MONGODB_URI="your-atlas-uri" npm run seed
MONGODB_URI="your-atlas-uri" npm run reset-admin
```

## 6. Fix "Not Found" on page reload (SPA routing)

If refreshing `/products`, `/orders`, etc. shows **Not Found**, add a rewrite rule on **each static site** (store + admin):

1. Render Dashboard → open **buylow-store** or **buylow-admin**
2. Go to **Redirects / Rewrites**
3. Add rule:
   - **Source:** `/*`
   - **Destination:** `/index.html`
   - **Action:** `Rewrite` (not Redirect)

If you deployed via **Blueprint**, sync the repo so `render.yaml` routes apply automatically.

The build also copies `index.html` → `404.html` as a fallback. Production apps use hash routing (`/#/orders`) so reload works even without dashboard rules.

## 7. Verify

- API health: `https://buylow-api.onrender.com/health`
- Store loads products
- Admin login: `admin@buylow.store` + your `ADMIN_PASSWORD`
- Reload on `/orders` or `/#/orders` does **not** show Not Found

## Notes

- **Uploaded images** on Render free tier use ephemeral disk — they reset on redeploy. Use external image URLs (Unsplash etc.) or add cloud storage later.
- Free Render services **sleep** after inactivity; first load may be slow.
- Always set `CLIENT_URL` with both store + admin URLs or CORS will block API calls.
